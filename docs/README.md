# Docs Index

Project docs are grouped by purpose so planning, status, setup, and design material are easy to find.

## Structure

- `architecture/`
  - high-level system direction and architecture overviews
- `audits/`
  - repo audits and review artifacts
- `design/`
  - engineering-facing design docs and onboarding references
- `handoffs/`
  - session handoffs and resume notes
- `meta/`
  - repo-specific AI/automation context and miscellaneous meta docs
- `planning/`
  - PRD, sprint plans, backlogs, trackers, and KPI review artifacts
- `research/`
  - interview guides and discovery material
- `setup/`
  - local environment and runtime setup guides
- `status/`
  - project status snapshots, local status notes, and history summaries
